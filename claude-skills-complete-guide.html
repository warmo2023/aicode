Claude Skills 是什么？

Claude Skills 是 Anthropic 在 2025年10月推出的新功能，让 Claude AI 能更高效地完成专业任务。很多人问：Skills 和之前的 MCP 有啥区别？到底该用哪个？

本文解决三个问题：Skills 是什么、能做什么、怎么用。

Skills 的组成

Skills 是一套模块化的能力包。每个 Skill 就是一个文件夹，里面包含：

  • SKILL.md 文件：说明这个 Skill 的用途和操作步骤
  • 脚本和模板：自动执行任务的代码（比如处理 Excel、生成 PPT）
  • 资源文件：辅助材料，像代码片段、流程文档等

Claude 会根据你的任务，自动判断需要哪些 Skills，然后加载使用。

打个比方，你给手机装了"美颜相机"App，拍照时打开就能用。Skills 就是给 Claude 装的专业工具包。

Skills 能做什么？效果如何？

Anthropic 实测数据

  • • 任务效率提升 40%
  • • 错误率下降 35%
  • • Token 使用量减少 14%-70%（Sonnet 3.7 模型）

为什么效果这么明显？因为 Skills 把常用任务封装好了，Claude 不用每次都"从头思考"怎么做，直接调用现成工具就行。

主要能力

1. 办公文档处理

生成 Excel 表格、制作 PPT、编辑 Word、处理 PDF。你给 Claude 一堆数据，它能直接生成格式规范的 Excel 报告。

2. 品牌和内容一致性

按照公司的风格指南写文案、设计素材，保证输出内容符合品牌调性。

3. 数据分析和处理

自动分析客户反馈、整理数据、生成可视化图表。

4. 可组合使用

多个 Skills 叠加。比如"品牌规范 Skill"保证文档风格，"报告生成 Skill"处理内容结构，"数据分析 Skill"做数据处理，三个一起用，完成复杂项目。

5. 跨平台通用

一套 Skills 可以在 Claude 应用、Claude Code、API 里通用，搭建一次到处用。

Skills vs MCP：区别在哪？

很多人搞混 Skills 和 MCP，其实它们职责不同。

核心区别

  • • Skills：教 Claude "怎么做"（流程、标准、方法）
  • • MCP：连接 Claude "到什么"（外部系统、数据库、API）

详细对比

维度 Claude Skills MCP（Model Context Protocol）
核心作用 教 Claude 执行任务的流程和标准 连接 Claude 到外部系统和数据源
适用场景 内部流程自动化（文档处理、风格规范、SOP） 外部系统集成（数据库、SaaS工具、API）
技术复杂度 简单（Markdown + 少量脚本） 复杂（完整协议规范、多传输方式）
Context 占用 每个 Skill 仅几十 token（仅加载元数据） 每个 MCP 服务器几千到上万 token
加载方式 渐进式：先加载元数据，需要时再加载完整内容 启动时加载完整内容
标准化程度 Anthropic 专属，但 OpenAI 已采用相同格式 开源协议，多厂商支持
典型用例 按风格指南写文案、生成报告、数据格式化 查询数据库、调用外部 API、访问云服务

举例说明

假设你要生成一份销售报告：

  • 用 Skills：Claude 按照公司报告模板（Skill 里定义的格式和风格）自动生成内容。
  • 用 MCP：Claude 先从数据库（通过 MCP）拉取销售数据，再生成报告。
  • Skills + MCP：MCP 负责拉取数据，Skills 负责按标准格式生成报告。

两者不是竞争，而是互补

  • • 如果任务是"怎么做"（流程、标准），用 Skills
  • • 如果任务需要"连接外部系统"（数据库、API），用 MCP
  • • 复杂场景可以两者结合：MCP 获取数据，Skills 处理数据

对普通用户日常工作，Skills 已经够用。企业级复杂场景（跨系统集成、自动分发），Skills + MCP 才是最佳组合。

如何安装和使用 Claude Skills？

官方 Skills 库

Anthropic 在 GitHub 开源了 12+ 个 Skills，涵盖：

  • 文档处理：PDF、Word、PPT、Excel
  • 创意设计：品牌规范、内容生成
  • 编程开发：代码测试、MCP 生成

GitHub 地址：github.com/anthropics/skills

安装方法

在 Claude Code 中执行这条命令：

/plugin marketplace add anthropics/skills

安装完成后，问 Claude："你有哪些 skills？"，就能看到可用的工具列表。

使用方式

你不需要手动选择 Skill，Claude 会自动判断。

比如你上传了一个 PDF 文件，要求提取表格数据，Claude 会自动调用"PDF 处理 Skill"，直接给出结果。

在 Claude 的思考过程（chain of thought）里，你能看到它调用了哪些 Skills。

使用限制

  • 支持用户：Pro、Max、Team、Enterprise 用户可用
  • API 使用：需要开启 files-api-2025-04-14 和代码执行工具的 beta 功能
  • 数量限制：每次请求最多加载 8 个 Skills

为什么要用 Claude Skills？对比传统方式

传统方式处理任务，Claude 需要临时生成代码、调试、修正，耗时且容易出错。

❌ 传统方式（不用 Skills）

  1. 上传 PDF 文件
  2. 告诉 Claude 提取表格
  3. Claude 现场生成 Python 脚本（用 pdfplumber 或 PyPDF2）
  4. 运行脚本，可能报错
  5. 修正代码，再运行
  6. 返回结果

整个过程多轮交互，消耗大量 token。

✅ 用 Skills

  1. 上传 PDF 文件
  2. 告诉 Claude 提取表格
  3. Claude 自动调用"PDF 处理 Skill"，运行预写好的脚本
  4. 几乎瞬时返回结果

任务标准化，处理流程固化，像"拿工具箱里的工具"一样快速。
