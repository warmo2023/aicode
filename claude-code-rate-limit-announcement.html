限流政策核心内容

2025年8月28日,Anthropic 正式实施 Claude Code 周限额政策,该政策同时影响:

受限订阅计划

  • Pro 订阅:20美元/月
  • Max 订阅:100美元/月 和 200美元/月

新政策引入两种周限额机制,每7天重置一次:

  1. 整体使用限额:覆盖所有模型的综合用量上限
  2. Opus 4 专项限额:针对最先进模型的独立额度

⚠️ 重要说明

周限额是新增限制,原有的5小时重置限额依然生效。这意味着用户现在需要同时应对两套限制机制,使用体验进一步收紧。

200美元Max订阅的具体限额

对于月付200美元的Max订阅用户,Anthropic 给出的周限额如下:

模型 每周限额
Claude Sonnet 4 240 - 480 小时
Claude Opus 4 24 - 40 小时

📊 限额计算方式说明

这里的"小时"并非实际时间,而是基于Token消耗量的虚拟计量单位:

  • 与代码库规模相关
  • 受对话长度影响
  • 取决于处理文本的复杂度

简单来说,实际可用时长因使用场景而异,并非字面意义上的工作时长。

Anthropic 还为Max订阅提供了补救措施:用户可以在达到周限额后,按标准API费率购买额外用量。但这种按需购买的成本远高于包月费用,对预算控制提出更高要求。

哪些用户会受到影响

根据 Anthropic 官方数据,新限额政策预计影响不到5%的现有订阅用户。但这一小部分用户恰恰是 Claude Code 的核心重度使用群体:

🚨 高风险用户

  • 24/7 Agent 运行:长期后台运行自动化任务的开发者
  • CI/CD 集成:将 Claude Code 嵌入持续集成流程的团队
  • 大规模代码生成:批量处理大型项目的专业用户
  • 账号共享/转售:违反服务条款的灰色使用行为

✅ 低风险用户

  • 常规开发:正常工作时间使用,偶尔请求AI辅助
  • 学习探索:学习编程或尝试新技术的初级用户
  • 轻量级任务:仅做代码审查、简单重构的轻度用户

💡 真实案例

据社交媒体流传的数据显示,有用户在20天内消耗了价值6,871美元的计算资源,远超200美元月费。这类极端用例正是 Anthropic 推出限额的直接原因。

Anthropic 为何推出限流政策

限流政策的推出并非偶然。Anthropic 官方给出的理由主要包括:

成本控制失衡

少数重度用户的月消耗远超订阅费用,导致 Anthropic 在这些账号上持续亏损。据内部消息,部分Max用户单月消耗数万美元计算资源,严重侵蚀平台利润率。

服务稳定性压力

过去一个月,Claude Code 经历了至少7次服务中断,主要原因是超预期的计算需求。限流政策旨在平衡负载,避免系统因少数用户过载而影响整体服务质量。

违规使用治理

部分用户通过账号共享、转售订阅等方式违规使用服务,导致单个账号承载多人需求。限额机制能有效遏制此类行为,保护正常用户权益。

从商业角度看,Anthropic 的限流政策是可以理解的。但对依赖 Claude Code 进行高强度开发的专业团队来说,这无疑是一次重大打击。好在市场上已有成熟的第三方服务可作为稳定替代。
