引言：为什么要掌握 Claude Code

第一部分：基础命令系统（功能 1-6）

功能 1：启动与退出 Claude Code

Claude Code 提供多种启动方式，适应不同的使用场景：

# 基本启动（交互模式）
claude

# 直接执行命令
claude "帮我创建一个 React 组件"

# 跳过权限确认（高级用户）
claude -c --dangerously-skip-permissions

# 退出 Claude Code
exit
# 或使用快捷键 Control + D

实战技巧：使用 claude -c 可以启用持续对话模式，避免每次都需要重新启动。

功能 2：版本管理与帮助系统

# 查看版本信息
claude --version

# 查看完整帮助文档
claude --help

# 查看特定命令的帮助
claude help /review

定期检查版本更新，Anthropic 持续优化 Claude Code 的性能和功能。

功能 3：工作目录管理

# 在指定目录启动 Claude Code
claude --directory /path/to/project

# 或先进入目录再启动
cd /path/to/project
claude

# 查看当前工作目录
pwd

最佳实践：始终在项目根目录启动 Claude Code，确保它能正确读取 Git 配置和 CLAUDE.md 文件。

功能 4：快捷键系统

掌握快捷键可以大幅提升操作效率：

编辑快捷键

  • • Control + L：清除屏幕，保留历史
  • • Control + C：取消当前操作
  • • Control + D：退出 Claude Code
  • • Option + Enter：换行（不发送）

功能快捷键

  • • Shift + Tab：开启 Plan 模式
  • • Up/Down 箭头：浏览历史命令
  • • Tab：命令自动补全
  • • Esc：取消当前输入

功能 5：模型选择与切换

# 启动时指定模型
claude --model claude-opus-4-5-20250914

# 在交互模式中切换模型
/model

# 常用模型选择
# - claude-opus-4-5：最强大，适合复杂任务
# - claude-sonnet-4-5：平衡性能与成本
# - claude-haiku-3-5：快速响应，简单任务

成本优化：对于简单的代码生成或问答，使用 Haiku 模型可以节省大量 API 费用。复杂的架构设计或重构任务则应使用 Opus 模型。

功能 6：上下文管理

# 查看当前上下文使用情况
/context

# 清除会话上下文
/clear

# 查看上下文窗口大小
/context --stats

Claude Code 自动管理上下文，但理解上下文限制可以帮助优化使用：

  • Opus 4.5：200,000 tokens 上下文窗口
  • Sonnet 4.5：200,000 tokens 上下文窗口
  • Haiku 3.5：200,000 tokens 上下文窗口

第二部分：核心斜杠命令（功能 7-14）

斜杠命令是 Claude Code 的核心功能，它们提供快速访问常用操作的方式。

功能 7：/init - 项目初始化

/init 命令为项目生成 CLAUDE.md 文件，这是 Claude Code 的"记忆系统"核心：

# 初始化项目，生成 CLAUDE.md
/init

# CLAUDE.md 文件会包含：
# - 项目概述
# - 技术栈信息
# - 开发规范
# - 特殊配置说明
# - 常见问题解答

CLAUDE.md 最佳实践

  • • 包含分支命名规范（feature/、bugfix/、hotfix/）
  • • 记录开发环境设置（Node 版本、环境变量等）
  • • 说明特殊的构建或测试流程
  • • 记录已知问题和解决方案

功能 8：/resume - 会话恢复

# 查看历史会话列表
/resume

# 恢复特定会话
# 选择会话编号后，所有上下文和对话历史都会恢复

# 会话管理技巧
# - 为长期项目保持活跃会话
# - 定期清理不需要的会话
# - 重要会话可以通过 /memory 添加关键信息

会话恢复功能让你可以在多个项目或任务之间无缝切换，保持上下文连贯性。

功能 9：/review - 智能代码审查

Claude Code 提供专业级别的代码审查功能：

# 审查当前暂存的更改
/review

# Claude 会检查：
# - 代码质量和最佳实践
# - 潜在的 bug 和安全问题
# - 性能优化建议
# - 代码风格一致性
# - 测试覆盖率

# 审查特定文件
/review src/components/Header.tsx

# 审查整个目录
/review src/

团队协作：在提交 PR 前使用 /review，可以提前发现问题，减少代码审查往返次数。

功能 10：/memory - 项目记忆管理

# 编辑 CLAUDE.md（项目记忆）

# 3. 本地记忆：.claude/CLAUDE.local.md（个人使用）

# 记忆文件会被递归读取
# 从当前目录向上搜索到根目录

  • • 架构决策：为什么选择某个技术栈或设计模式
  • • 常见问题：已知 bug、解决方案、调试技巧

第三部分：高级自动化功能（功能 15-20）

功能 15：Subagents - 并行任务处理

Subagents 是 Claude Code 的革命性功能，允许将任务委派给专门的子代理并行执行：

# 示例：同时开发前后端
"请创建一个全栈应用：
- 子代理1：构建 Express API 后端
- 子代理2：同时开发 React 前端
- 确保两者的接口定义一致"

# Subagents 工作原理
# 1. 主代理分析任务，识别可并行的部分
# 2. 创建专门的子代理处理各个子任务
# 3. 子代理独立工作，不会相互干扰
# 4. 主代理协调所有子代理的输出

# 适用场景
# - 前后端同步开发
# - 多模块功能实现
# - 大型重构项目
# - 并行测试编写

性能提升：使用 Subagents，复杂项目的开发时间可以缩短 50% 以上。

功能 16：Hooks - 自动化工作流钩子

Hooks 在特定事件发生时自动触发操作，实现真正的自动化工作流：

# 配置 Hooks（在 .claude/settings.json 中）
{
  "hooks": {
    "pre-commit": [
      "npm run lint",
      "npm test"
    ],
    "post-code-change": [
      "npm run format",
      "/review"
    ],
    "pre-push": [
      "npm run build",
      "npm run test:e2e"
    ]
  }
}

# 常用 Hook 类型
# - pre-commit：提交前执行（代码检查、测试）
# - post-commit：提交后执行（通知、部署）
# - pre-push：推送前执行（集成测试、构建验证）
# - post-code-change：代码变更后（格式化、自动审查）
# - on-error：错误发生时（日志记录、回滚）

Hooks 实战案例

完整的质量保证流程：

  • 1. 代码变更后自动运行 Prettier 格式化
  • 2. 提交前自动执行 ESLint 和单元测试
  • 3. 推送前运行完整测试套件和构建
  • 4. 所有步骤失败时自动回滚

功能 18：Checkpointing - 任务检查点系统

Checkpointing 让你可以安全地委派复杂任务，同时保持完全控制：

# 启用检查点模式
"使用检查点模式重构用户认证模块"

# 工作流程
# 1. Claude 分析任务，创建执行计划
# 2. 在关键步骤设置检查点
# 3. 每个检查点需要用户确认才能继续
# 4. 可以随时回滚到之前的检查点

# 检查点触发时机
# - 即将修改核心代码
# - 需要删除文件或代码
# - 进行数据库迁移
# - 修改配置文件

# 检查点操作
approve     # 批准继续
modify      # 修改计划
rollback    # 回滚到上一个检查点
cancel      # 取消整个任务

安全保障：Checkpointing 特别适合重构、数据迁移等高风险操作。

功能 19：MCP - 扩展 Claude 的能力边界

Model Context Protocol (MCP) 让 Claude Code 能够与外部工具和服务安全交互：

# MCP 配置（.claude/mcp.json）
{
  "servers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "your-token"
      }
    },
    "database": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "DB_URL": "postgresql://localhost/mydb"
      }
    }
  }
}

# 通过 MCP 使用 GitHub
"查询仓库 user/repo 中最近关闭的 10 个 issue"
"创建一个新 issue 报告这个 bug"
"审查 PR #123 的代码变更"

# 通过 MCP 使用数据库
"查询 users 表中最近注册的用户"
"创建数据库迁移脚本"

可用的 MCP 服务器

  • • GitHub：issue、PR、仓库管理
  • • 数据库：PostgreSQL、MySQL、MongoDB
  • • 文件系统：高级文件操作
  • • Web API：调用外部 API
  • • 自定义服务器：扩展任何功能

技巧 4：成本优化策略

按任务选择模型：
  • - Haiku：代码格式化、简单 bug 修复、文档生成
  • - Sonnet：功能开发、代码审查、测试编写
  • - Opus：架构设计、复杂重构、难度调试
批量操作：

将多个小任务合并成一个请求，减少 API 调用次数

上下文管理：
