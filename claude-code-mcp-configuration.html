一、配置 MCP 前的准备工作

环境要求检查

开始配置之前,先确认这些软件都装好了:

必需软件

  • Claude Code CLI 已安装(版本 >= 1.0)
  • Node.js >= 18.0(MCP 服务器的运行环境)
  • npm 或 yarn(包管理器)
验证安装
# 验证 Claude Code
claude --version

# 验证 Node.js
node --version  # 应显示 v18.x 或更高

# 验证 npm
npm --version

MCP 配置文件结构

MCP 配置文件位置:

macOS/Linux

~/.claude/mcp.json
或
~/Library/Application Support/Claude/claude_desktop_config.json

Windows

%APPDATA%\Claude\claude_desktop_config.json

基础配置文件结构:

{
  "mcpServers": {
    "服务器名称": {
      "command": "执行命令",
      "args": ["参数1", "参数2"],
      "env": {
        "环境变量名": "环境变量值"
      }
    }
  }
}

二、Claude Code 配置 MCP 的三种方法

方法 1: 使用 CLI 命令配置(推荐新手)

CLI 命令最方便,自动处理配置文件格式,还会实时验证配置是否正确。

添加 GitHub MCP 服务器

# 使用 CLI 配置 GitHub MCP
claude mcp add github --scope user

执行后会提示输入 GitHub Personal Access Token,输入完成后自动写入配置文件。验证配置:

# 列出所有已配置的 MCP 服务器
claude mcp list

# 输出示例:
github (running)
├─ Type: npm
└─ Package: @modelcontextprotocol/server-github

添加 PostgreSQL MCP 服务器

# 配置 PostgreSQL MCP
claude mcp add postgres \
  -e POSTGRES_URL="postgresql://user:pass@localhost:5432/db"

CLI 配置的优势

  • ✅ 自动处理配置文件格式,不用担心 JSON 语法错误
  • ✅ 实时验证配置有效性
  • ✅ 避免手动编辑时的拼写错误

方法 2: 直接编辑配置文件(适合批量配置)

对于需要同时配置多个 MCP 服务器的场景,直接编辑配置文件更高效。

创建配置文件

如果配置文件不存在,先创建:

# macOS/Linux
mkdir -p ~/.claude
touch ~/.claude/mcp.json

完整配置示例(多服务器)

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_TOKEN}"
      }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "POSTGRES_URL": "${DATABASE_URL}"
      }
    },
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname/projects"
      ]
    }
  }
}

配置说明

  • command: MCP 服务器的启动命令
  • args: 传递给命令的参数
  • env: 环境变量(推荐使用 ${VAR} 引用系统环境变量)

方法 3: 使用 JSON 配置命令(高级用户)

通过 CLI 直接传递 JSON 配置,适合快速测试或脚本自动化:

# 配置自定义 HTTP MCP 服务器
claude mcp add-json weather-api '{
  "type": "http",
  "url": "https://api.weather.com/mcp",
  "headers": {
    "Authorization": "Bearer ${WEATHER_API_TOKEN}"
  }
}'

三、高级配置技巧

技巧 1: 使用环境变量管理敏感信息

❌ 错误的配置方式(不安全)

{
  "mcpServers": {
    "postgres": {
      "env": {
        "POSTGRES_URL": "postgresql://admin:[email protected]:5432/users"
      }
    }
  }
}

密码明文写在配置文件里,极度不安全!

✅ 正确的配置方式(安全)

Step 1: 配置系统环境变量

# macOS/Linux - 添加到 ~/.zshrc 或 ~/.bashrc
export POSTGRES_URL="postgresql://admin:[email protected]:5432/users"
export GITHUB_TOKEN="ghp_your_github_token_here"

# 重新加载配置
source ~/.zshrc

Step 2: 配置文件引用环境变量

{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "POSTGRES_URL": "${POSTGRES_URL}"
      }
    }
  }
}

安全优势

  • ✅ 配置文件不包含明文密码
  • ✅ 可以安全提交到 Git(配合 .gitignore)
  • ✅ 不同环境(开发/生产)使用不同环境变量

技巧 2: 配置 MCP 服务器超时时间

默认启动超时是 5 秒。启动慢的服务器需要增加超时:

# 设置 MCP 超时为 10 秒
export MCP_TIMEOUT=10000

# 或在配置文件中设置(推荐)
{
  "mcpServers": {
    "slow-server": {
      "command": "npx",
      "args": ["-y", "@your-org/slow-mcp-server"],
      "timeout": 15000
    }
  }
}

技巧 3: 配置 MCP 输出 Token 限制

MCP 服务器返回的数据量可能很大,默认限制是 20,000 tokens。如果需要处理更大的数据集:

# 设置为 50,000 tokens
export MAX_MCP_OUTPUT_TOKENS=50000

适用场景:

  • 数据库查询返回大量数据
  • 代码库搜索结果很多
  • 文件系统读取大文件

四、实战案例

案例 1: 配置企业级 GitHub MCP

需求: 连接企业 GitHub Enterprise Server

配置文件
{
  "mcpServers": {
    "github-enterprise": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GHE_TOKEN}",
        "GITHUB_API_URL": "https://github.your-company.com/api/v3"
      }
    }
  }
}
环境变量配置
export GHE_TOKEN="ghp_your_enterprise_token"
验证配置
claude mcp list
# 在 Claude Code 中测试
# 输入: @github-enterprise/issues

案例 2: 配置多数据库 MCP 连接

需求: 同时连接 PostgreSQL、MySQL、MongoDB

{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "POSTGRES_URL": "${PG_URL}"
      }
    },
    "mysql": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-mysql"],
      "env": {
        "MYSQL_URL": "${MYSQL_URL}"
      }
    },
    "mongodb": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-mongodb"],
      "env": {
        "MONGODB_URI": "${MONGO_URI}"
      }
    }
  }
}

使用示例:

在 Claude Code 中:

  • "@postgres 查询 users 表的前 10 条记录"
  • "@mysql 查询订单总数"
  • "@mongodb 查找最近的日志记录"

五、故障排查

常见错误 1: MCP 服务器启动失败

Error: MCP server 'github' failed to start

排查步骤:

Step 1: 检查 Node.js 版本

node --version
# 必须 >= 18.0

Step 2: 手动测试 MCP 服务器

npx @modelcontextprotocol/server-github
# 查看是否有错误输出

Step 3: 检查环境变量

echo $GITHUB_PERSONAL_ACCESS_TOKEN
# 确保已设置

Step 4: 查看日志

# macOS/Linux
cat ~/.claude/logs/mcp.log

# 实时查看
tail -f ~/.claude/logs/mcp.log

常见错误 2: 配置文件语法错误

Error: Invalid MCP configuration format

解决方案: 使用 jq 验证 JSON 格式

# 验证 JSON 格式
cat ~/.claude/mcp.json | jq .

# 如果有语法错误,jq 会报错并指出位置

常见语法错误

  • ❌ 缺少逗号
  • ❌ 多余的逗号(最后一项后)
  • ❌ 引号不匹配
  • ❌ 环境变量格式错误

常见错误 3: 网络问题导致 MCP 安装失败

六、CLI 命令完全参考

基础命令

列出所有 MCP 配置

claude mcp list

查看特定 MCP 服务器详情

claude mcp get github

删除 MCP 配置

claude mcp remove github

在 Claude Code 中检查 MCP 状态

高级命令

添加带作用域的 GitHub MCP

# 用户作用域
claude mcp add github --scope user

# 组织作用域
claude mcp add github --scope org:your-org

添加带环境变量的 MCP

claude mcp add digitalocean-mcp-local \
  -e DIGITALOCEAN_API_TOKEN=YOUR_TOKEN \
  -- npx "@digitalocean/mcp"

重启 MCP 服务器

# 重启单个服务器
claude mcp restart github

# 重启所有服务器
claude mcp restart --all
