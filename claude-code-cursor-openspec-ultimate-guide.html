8分钟完成一次性配置

第一步：全局安装（2分钟）

# 安装Claude Code和OpenSpec
npm install -g @anthropic-ai/claude-code @fission-ai/openspec

# 进入项目目录
cd your-project

第二步：初始化OpenSpec（3分钟）

# 初始化规范目录
openspec init

# 自动生成以下结构：
# openspec/
# ├── project.md # 全局项目规范
# ├── AGENTS.md # AI角色分工规则
# └── changes/ # 变更提案目录

第三步：配置Claude Code（2分钟）

# 启动Claude Code
claude

# 在终端中执行配置
/config auto-accept-changes true
/load openspec/AGENTS.md
/load openspec/project.md

5步核心工作流（日常开发必备）

第1步：OpenSpec创建变更提案（3-8分钟）

# 创建新需求
openspec new add-user-rbac

# 填写4个必填部分：
# • Requirements - 功能需求描述
# • Constraints - 技术约束和限制
# • Design - 设计方案（可让Claude先起草）
# • Tasks - 自动拆分1-8个子任务

第2步：Claude Code生成实施计划（5-15分钟）

在Claude终端输入：

/plan @openspec/changes/add-user-rbac.md
Think ultrahard, output step-by-step implementation plan

Claude会输出详细的：

  • 文件修改清单
  • 代码实现顺序
  • 依赖关系分析
  • 潜在风险提示

第3步：Cursor快速执行（10-40分钟）

  1. 复制Claude的计划到Cursor Composer（快捷键Cmd+K）
  2. 使用Tab键让Cursor自动补全代码
  3. 遇到Bug时：
    • 截图错误 → 粘贴给Claude Code分析
    • 或使用Cursor内置的@bugbot快速定位

第4步：Claude Code终审与提交（5-10分钟）

# 自动审查所有变更
/review all changes against @openspec/changes/add-user-rbac.md

# 通过后自动归档并创建PR
If pass → /openspec:archive add-user-rbac --yes
Then create PR with conventional commits

黄金Prompt合集（实战验证有效）

1. 规划神器Prompt

Plan mode: 基于 @openspec/changes/xxx.md，输出4-8个线性子任务，
每个任务包含：
• 任务描述（1句话）
• 涉及文件路径
• 预估时间
• 优先级（P0/P1/P2）
Think ultrahard, 确保任务间无依赖冲突。

2. 执行神器Prompt

Implement: 严格按照 @openspec/changes/xxx.md 的 Design 部分实现当前任务。
要求：
• 单个文件不超过300行
• 不引入新依赖
• 完成后自动运行: pnpm test && pnpm lint

3. 审查神器Prompt

Review: 交叉检查所有 git diff 是否100%符合：
1. openspec/project.md 全局规范
2. openspec/changes/xxx.md 具体要求

评分标准：0-10分
如果 < 9分，列出具体不符合项和修改建议。

4. 一键归档Prompt

/openspec:archive xxx --yes && gh pr create --fill

常见坑与避坑指南

坑1：AI疯狂添加新依赖

问题现象：Claude或Cursor动不动就npm install新包，导致项目臃肿

解决方案：

# 在 project.md 中写死依赖白名单
## 允许的依赖包
- UI: @radix-ui/react-*
- 工具: lodash, dayjs
- 验证: zod

## 严格禁止
添加任何未在白名单中的依赖，必须先提issue讨论。

坑2：代码风格不统一

问题现象：Claude用2空格，Cursor用4空格，团队协作混乱

解决方案：

  1. Cursor Rules文件同步OpenSpec规则
  2. 配置.prettierrc和.eslintrc
  3. Git pre-commit hook强制格式化

坑3：后期改需求导致大重构

问题现象：客户临时加需求，牵一发动全身

解决方案：

  • 每个新需求都必须openspec new创建提案
  • 评估影响范围 → Claude Code出方案 → 评审后再动手
  • 拒绝口头需求，一切以OpenSpec文档为准

坑4：Claude Code卡死或超时

问题现象：执行复杂任务时Claude终端无响应

解决方案：

# 立即 Ctrl+C 终止
# 新开终端重新加载
claude
/clear
/load openspec/project.md
/load openspec/AGENTS.md
# 继续之前的任务

坑5：Cursor一直改不相关文件

问题现象：让Cursor改A文件，结果把B、C、D都改了

解决方案：

  • Composer中先选中目标文件
  • 勾选"Only modify files in tasks"选项
  • 在Prompt中明确指定文件路径
