一、CLAUDE.md：Claude 的项目记忆系统

1.1 什么是 CLAUDE.md？

CLAUDE.md 是 Claude Code 启动时自动加载的配置文件，相当于给 Claude 配备了一个项目知识库。通过这个文件，你可以让 Claude 记住项目的代码规范、技术栈、开发流程等关键信息。

基础 CLAUDE.md 示例 

# 我的项目

这是一个 React 项目，使用 TypeScript 开发。

## 代码规范
- 组件使用函数式写法
- 状态管理使用 Zustand
- 样式使用 Tailwind CSS

## 注意事项
- 不要使用 class 组件
- 提交前运行 eslint 检查

1.2 CLAUDE.md 的三级配置体系

Claude Code 支持三个级别的 CLAUDE.md 配置，按优先级从低到高：

级别 文件路径
全局配置 ~/.claude/CLAUDE.md 通用代码风格、个人偏好
项目配置 ./CLAUDE.md 项目特定规范、团队约定
子目录配置 ./[子目录]/CLAUDE.md 模块特定规则

实用技巧： 在对话中按 # 键可以快速创建或编辑 CLAUDE.md。Claude 会根据你的输入自动更新配置文件。

1.3 使用 /init 命令自动生成

Claude Code 提供了 /init 命令，可以自动分析项目结构、技术栈和依赖，生成针对性的 CLAUDE.md 配置。实测显示，自动生成的配置 80% 的内容可以直接使用。

二、Commands：自定义快捷指令

2.1 什么是 Commands？

Commands 允许你创建自定义的快捷指令。在 .claude/commands/ 目录下创建 markdown 文件，就能定义专属命令。

创建 Commands 示例 

# 文件名：.claude/commands/write-article.md

帮我写一篇技术文章，主题是：$ARGUMENTS

要求：
1. 专业技术风格
2. 字数 2000-3000
3. 保存到 articles 目录
4. 包含代码示例

使用方式：/write-article Claude Code 教程

2.2 Commands 的局限性

注意： Commands 不支持条件判断和复杂逻辑。对于需要多步骤、有条件分支的任务，建议使用 Skills（下一节介绍）。

三、Skills：智能技能模块系统

3.1 Skills 与 Commands 的区别

如果说 Commands 是简单的命令别名，那么 Skills 就是具备完整工作流程的智能模块。Skills 支持多步骤执行、工具调用权限管理、条件分支等高级特性。

特性 Commands Skills
复杂度 简单指令 多步骤工作流
工具权限 无法限制 精确控制（allowed-tools）
条件分支 不支持 完全支持

3.2 创建 Skills 的完整结构

Skills 目录结构 

.claude/skills/
├── article-writer/
│   ├── SKILL.md          # 技能定义（必须）
│   ├── instructions/     # 详细指令
│   ├── prompts/          # 提示词模板
│   ├── templates/        # 代码/文档模板
│   └── config/           # 配置文件

SKILL.md 示例 

---
name: article-writer
description: 技术文章写作助手
allowed-tools: WebSearch, Read, Write
---

# 文章写作流程

## 步骤1：搜索最新资料
使用 WebSearch 搜索主题相关的最新信息

## 步骤2：读取写作规范
Read(".claude/skills/article-writer/style-guide.md")

## 步骤3：写作
基于搜索结果和规范，写一篇 2500 字的文章

## 步骤4：保存
Write 到 articles 目录，文件名使用主题关键词

3.3 Skills 的智能加载机制

Claude 不会一股脑把所有 Skills 都加载进来，而是根据当前任务动态选择相关的 Skill。这种上下文驱动的加载方式既节省 token，又提高了响应速度。

实测经验： 一个完善的写作 Skill 可以达到 500+ 行，包含多个子步骤和条件判断。通过 allowed-tools 精确控制权限后，可以避免 Claude 调用不必要的工具，显著提升执行效率。

四、MCP（Model Context Protocol）：无限扩展 Claude 能力

4.1 什么是 MCP？

Model Context Protocol（MCP）是 Anthropic 推出的标准化协议，允许第三方开发者为 Claude Code 开发扩展功能。通过 MCP，你可以让 Claude 调用浏览器、数据库、API 等外部工具。

MCP 服务器配置示例 

{
  "mcpServers": {
    "chrome-devtools": {
      "command": "npx",
      "args": ["@anthropic/mcp-chrome-devtools"]
    },
    "postgres": {
      "command": "npx",
      "args": ["@anthropic/mcp-postgres"],
      "env": {
        "DATABASE_URL": "postgresql://localhost/mydb"
      }
    }
  }
}

配置文件路径：.claude/settings.json

4.2 MCP 的优势与挑战

MCP 优势

  • ✅ 能力扩展无限，Claude 做不到的都能实现
  • ✅ 标准化协议，一次配置到处可用
  • ✅ 社区资源丰富，常用功能有现成方案
  • ✅ 热插拔，按需启用/关闭

注意事项

  • ⚠️ Token 消耗增加（工具列表占用）
  • ⚠️ 响应速度变慢（等待各服务器响应）
  • ⚠️ 内存问题（部分服务器有泄漏）
  • ⚠️ 配置门槛（需要懂 JSON 和命令行）

调试技巧： 使用 --mcp-debug 标志启动 Claude Code，可以查看 MCP 服务器的详细日志，快速定位配置问题。

五、Hooks：自动化守护与流程控制

5.1 Hooks 的核心价值

Hooks 允许你在 Claude Code 执行的关键节点插入自定义脚本，实现安全守护、自动化处理、质量检查等功能。这是保证 AI 编程可靠性的重要机制。

Hooks 配置示例 

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hooks": ["python .claude/hooks/check_before_write.py"]
      }
    ],
    "SessionStart": [
      {
        "hooks": ["python .claude/hooks/backup_context.py"]
      }
    ]
  }
}

5.2 常用 Hooks 类型

SessionStart

会话开始时触发。用途：保存重要上下文，防止压缩丢失关键信息。

PreToolUse

工具调用前触发。用途：安全检查，拦截危险操作（如删除重要文件）。

UserPromptSubmit

用户提交提示后触发。用途：自动添加项目上下文，增强 prompt 质量。

SubagentStop

子代理停止时触发。用途：整合多个子代理的结果，生成汇总报告。

实战案例： 配置了一个 PreToolUse Hook 专门检查 Claude 要写的代码质量。在文件保存前自动运行 ESLint，发现问题立即拦截并提示修复，避免了低质量代码进入仓库。

六、Subagents：并行处理复杂任务

6.1 Subagents 的工作机制

Subagents（子代理）允许你启动多个独立的 Claude 实例，并行处理不同的子任务。每个子代理拥有独立的上下文，互不干扰，适合处理复杂的多步骤项目。

Subagents 调用示例 

# 基础调用
Task(
  subagent_type="Explore",
  prompt="搜索项目里所有的 API 端点"
)

# 后台运行模式
Task(
  subagent_type="Explore",
  prompt="搜索所有 API 端点",
  run_in_background=true
)

6.2 何时使用 Subagents？

适合使用 Subagents 的场景：

  • 需要同时搜索、分析和编写代码的复杂项目
  • 大型重构任务，需要并行处理多个模块
  • 数据处理 + 分析 + 可视化的组合任务
  • 需要配合 ultrathink 模式进行深度思考的问题

Subagents 的使用成本：

  • ⚠️ Token 爆炸： 每个子代理独立消耗 token，开3个就是3倍消耗
  • ⚠️ 协调复杂： 多个子代理结果需要整合，可能产生冲突
  • ⚠️ 调试困难： 出问题不知道是哪个子代理的锅
  • ⚠️ 启动开销： 简单任务反而更慢

建议策略： 复杂任务才值得开子代理。对于需要长期大量使用 Subagents 功能的开发者，通过 GACCode 订阅可以获得更稳定的 token 配额和更高的并发支持，避免官方账户在高负载场景下的降额风险。

七、Plugins：一站式扩展生态

7.1 Plugins 是什么？

Plugins 是 Claude Code 的扩展插件生态，打包了 Commands、Agents、Skills、Hooks、MCP 服务器等多种组件。一个插件可以包含完整的工作流解决方案。

安装 Plugins 

# 安装插件
/plugin add anthropic-agent-skills

# 使用插件提供的功能
document-skills:pdf    # 处理 PDF
document-skills:xlsx   # 处理 Excel
document-skills:pptx   # 处理 PPT

7.2 推荐的官方 Plugins

📝 document-skills

处理各类文档格式（PDF、Excel、PPT），自动提取内容、转换格式、生成摘要。

适用场景： 文档批量处理、内容分析

🔍 code-review

自动化 PR 代码审查，检查代码质量、安全漏洞、性能问题，生成详细报告。

适用场景： 团队协作、质量保证

🚀 commit-commands

Git 工作流自动化，一键完成提交+推送+创建 PR，自动生成符合规范的 commit message。

适用场景： 规范化 Git 流程

🛡️ security-guidance

安全提醒 Hook，在代码中检测潜在安全漏洞（SQL 注入、XSS 等），提供修复建议。

适用场景： 安全敏感项目

实测体验： 官方的 document-skills 插件真香！可以一键提取 PDF 表格数据，转换成 Excel 格式，省去了大量手动操作。配合 Skills 工作流，还能自动生成数据分析报告。
